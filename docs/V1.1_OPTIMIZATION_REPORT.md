## 二、新增模块

### 2.1 Hooks 自动配置系统

**文件**: `src/harnessgenj/harness/hooks_auto_setup.py`

**功能**:
- 框架初始化时自动检测 Hooks 配置状态
- 未配置时自动创建 `.claude/settings.json` 和 `harnessgenj_hook.py`
- 支持静默模式和强制更新

**使用**:
```python
from harnessgenj import Harness

# 初始化时自动配置 Hooks
harness = Harness("my-project")

# 或手动触发
from harnessgenj.harness import auto_setup_hooks
auto_setup_hooks("/path/to/project")
```

---

### 2.2 技术栈自动检测系统

**文件**: `src/harnessgenj/harness/tech_detector.py`

**功能**:
- 自动检测项目技术栈（语言、框架、构建工具、数据库等）
- 根据检测结果生成适配的 `agents/tech.md` 和 `agents/conventions.md`
- 支持 Python、Java/Android、Node.js、Go、Rust、Flutter 等

**检测内容**:
- 主要语言
- 框架列表
- 构建工具
- 包管理器
- 数据库
- 测试框架
- 部署方式

**使用**:
```python
from harnessgenj.harness import detect_tech_stack, update_agents_templates

# 检测技术栈
tech_info = detect_tech_stack("/path/to/project")
print(f"Language: {tech_info.main_language}")
print(f"Frameworks: {tech_info.frameworks}")

# 更新模板
update_agents_templates(".harnessgenj", tech_info)
```

### 2.3 结构化知识管理器

**文件**: `src/harnessgenj/memory/structured_knowledge.py`

**功能**:
- 为知识条目添加唯一 ID、类型分类、代码位置索引
- 支持按类型、标签、文件检索知识
- 嵌入到 MemoryManager 中，统一接口访问

**知识类型**:
- `bug_fix` - Bug 修复记录
- `decision_pattern` - 决策模式沉淀
- `architecture_change` - 架构演进追踪
- `security_issue` - 安全问题追踪
- `test_case` - 测试用例库

**使用**:
```python
from harnessgenj.memory import KnowledgeEntry, KnowledgeType

# 创建知识条目
entry = KnowledgeEntry(
    type=KnowledgeType.SECURITY_ISSUE,
    problem="ShellTool命令注入风险",
    solution="使用命令白名单模式",
    code_location={"file": "app/ShellTool.java", "lines": [93, 118]},
    severity="critical",
    tags=["security", "shell"],
)

# 存储
harness.memory.store_structured_knowledge(entry)

# 查询
issues = harness.memory.query_knowledge_by_type(KnowledgeType.SECURITY_ISSUE)
```

### 2.4 事件触发管理器

**文件**: `src/harnessgenj/harness/event_triggers.py`

**功能**:
- 事件驱动的角色激活机制
- 预定义触发规则（Write 完成 → CodeReviewer，任务完成 → BugHunter）
- 支持自定义触发规则和条件

**触发矩阵**:
| 事件 | 触发角色 |
|------|----------|
| Write 完成 | CodeReviewer |
| Edit 完成 | CodeReviewer |
| 任务完成 | BugHunter, Tester |
| 安全问题发现 | CodeReviewer, BugHunter |
| 架构变更 | Architect |

**使用**:
```python
from harnessgenj.harness import TriggerEvent

# 触发事件
harness._trigger_manager.trigger(
    TriggerEvent.ON_WRITE_COMPLETE,
    {"file_path": "src/main.java", "content": "..."},
)

# 注册自定义规则
harness._trigger_manager.register_trigger(
    event=TriggerEvent.ON_WRITE_COMPLETE,
    role_types=["bug_hunter"],
    condition=lambda ctx: "security" in ctx.get("content", ""),
)
```

## 三、架构改进

### 3.1 嵌入式设计

所有新模块均嵌入到现有架构中，不增加冗余：

```
MemoryManager
├── heap (MemoryHeap) - JVM 分代堆
├── gc (GarbageCollector)
├── hotspot (HotspotDetector)
└── structured (StructuredKnowledgeManager) [嵌入]

Harness
├── memory (MemoryManager)
├── coordinator (WorkflowCoordinator)
├── _hooks_integration (HooksIntegration)
├── _trigger_manager (TriggerManager) [嵌入]
└── ...
```

### 3.2 统一接口

新功能通过现有接口访问，保持 API 简洁：

```python
# 结构化知识 - 通过 memory 访问
harness.memory.store_structured_knowledge(entry)
harness.memory.query_knowledge_by_type(type)

# 事件触发 - 内部自动调用
harness.develop(...)  # 完成后自动触发 ON_TASK_COMPLETE

# 技术栈检测 - from_project 时自动调用
harness = Harness.from_project(".")  # 自动检测并更新模板
```
